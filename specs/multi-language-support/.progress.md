## Design Learnings (2026-05-22)
- `set -euo pipefail` placement DECIDED: inside the BASH_SOURCE main-guard (not top-level, not in function). Top-level leaks `set -e` into the orchestrator on source (would abort the loop on any non-zero); function-scope risks leaking `pipefail` to caller on return. Guard is cleanest and gives CLI strict mode.
- `SPEC_PATH`/`ENTRIES`/`FILTERED` MUST be `local` to `detect_ci_commands` → re-callable + zero global pollution of sourcing shell (AC-9.3). Detectors see ENTRIES via bash dynamic scope (called from within the function). Detectors stay top-level (defined at source time, exist in both modes).
- Filter `./` patch: `if [[ "$bin" == ./* ]]; then [[ -x "$SPEC_PATH/$bin" ]] || keep=0; else command -v ...; fi`. SPEC_PATH is the function's local arg → in scope inside the filter. `$SPEC_PATH/./gradlew` (the `/./`) is harmless.
- detect_dotnet: glob markers via `compgen -G "$base/*.csproj"` wrapped in if/||-chain so no-match never aborts; global.json stays `[[ -f ]]`. Confirmed schema enum is {lint,typecheck,test,build,other} (spec.schema.json:658).
- Confirmed quality-commands.md ALREADY has Ruby(69)/JVM(70)/Elixir(71)/Deno(72) rows → add ONLY PHP + C#/.NET (2 rows).
- NFR-5 caveat: `shellcheck` NOT installed in this env; `bash -n` available + clean. Existing syntax test uses bash -n. Enforced gate = bash -n clean; shellcheck = manual/CI-side. Documented as Unresolved Question, not a blocker.
- Testing discovery: bats present, 17/17 legacy pass (smoke clean). No new fixture FILES — tests build markers inline in temp dirs + extend STUBBIN loop with composer/bundle/mix/deno/dotnet/gradle/mvn. PATH binaries are STUBS not MOCKS (we assert JSON output, not invocation).
- Maven canonical build = `mvn package` (`./mvnw package` w/ wrapper); Gradle = test+build only, NO check-as-typecheck (AC-3.4 honest categories).
